# 200 OK — 加強版引導式「建立新專案」設計方案（Client 友善版）

🎯 **目標：** 幫助對軟體開發完全不熟悉的發案者，也能在無需懂技術詞彙的情況下，清楚說出需求。透過「引導式對話 + 智能補全」，自動轉化成清晰可估工時的 Project Brief。

## 一、整體策略

- 以「像聊天一樣」的問答形式收集需求。
- 問題語言不使用技術名詞，改用生活語言＋具體行為描述。
- 提供常見選項（ERP、官網、電商、LineBot、App、遊戲…），但當使用者選「其他」時，系統進一步引導補問。
- 後端自動轉譯成結構化 brief（JSONB），供接案者直接理解需求。

## 二、主要引導問題設計

### Step 1｜你想做的東西屬於哪一類？

**問法：** 「你想做的軟體大概是哪種類型？」

**常見分類卡片式選項：**

- 🏢 官方形象網站（公司介紹、聯絡表單）
- 🛒 電商平台（商品展示、購物車、金流）
- 🧾 ERP / CRM 系統（內部管理用）
- 💬 LineBot / Chatbot（自動客服、行銷工具）
- 📱 手機 App（會員系統、預約系統）
- 🎮 手機遊戲 / 網頁遊戲
- 🧩 其他（需說明）

**若選「其他」** → 顯示下一題：
- 「能用一句話說說你想做什麼嗎？」
- （例：『想做一個幫學生記錄學習進度的小工具』）

### Step 2｜使用情境（讓我們更懂你的畫面）

**問法：** 「你想像這個軟體會被怎麼使用？」

**範例回答：**

- 客人打開手機 → 預約剪頭髮 → 收到提醒
- 員工登入 → 填報表 → 自動算薪水
- 老師輸入學生資料 → 自動產生成績圖表

**輔助按鈕：**

- 🔍 看範例畫面（同類型常見介面）

### Step 3｜目標與成果

**問法：** 「這個軟體主要想幫你『解決什麼問題』或『達成什麼目的』？」

**範例：**

- 讓顧客更快預約 / 查詢資訊
- 減少重複手動作業
- 提升銷售轉換率
- 自動統計數據報表

✅ **小提示：** AI 會根據這題自動生成專案摘要。

### Step 4｜你希望使用者可以做哪些事？

**問法：** 「想像你是用戶，打開這個軟體後你希望他能完成哪些事？」

**支援清單式輸入，每項一句：**

- 「瀏覽商品」
- 「上傳作品」
- 「線上報名」

**AI 建議按鈕：** 「幫我自動列出常見功能」

### Step 5｜你自己要看到什麼？（輸出需求）

**問法：** 「軟體完成後，你希望自己能在哪裡看到成果或數據？」

**選項：** 後台名單 / 銷售報表 / Email 通知 / Excel 匯出

**若選「其他」** → 「能舉例你想要的畫面嗎？」

### Step 6｜有沒有想參考的網站或 App？

**問法：** 「有沒有你覺得長得不錯的網站或 App？」

**欄位：** 可貼連結 / 上傳截圖 / 用幾個形容詞描述（簡潔、溫暖、科技感…）

**若無** → AI 自動推薦 3 種風格拼盤（企業感 / 親切型 / 活潑型）

### Step 7｜有沒有特別要搭配的工具？

**問法：** 「這個系統需要跟其他工具連動嗎？」

**常見選項：** Google Sheet / Line / FB / CRM / 金流 / 會員資料庫

**若有** → 「可提供帳號或 API 文件嗎？」

**若無** → 「那我們會幫你選最方便的方式。」

### Step 8｜預算與時程

**問法：** 「希望什麼時候能完成？預算大概在哪個範圍？」

**預算選擇（拉桿）：**

使用拉桿選擇預算區間：

```
最低預算 ──●─────────────── 最高預算
  $10k    $50k    $100k   $200k   $500k+
```

- **操作方式：** 拖曳兩個拉桿點設定預算範圍（最低預算 ~ 最高預算）
- **預設區間：** $40k–$80k
- **範圍選項：** $10k–$500k+（可自訂）
- **即時顯示：** 拉桿下方即時顯示選擇的預算區間

**其他選項：**
- ⚠️ 先估型（不確定預算，希望接案者報價）


**時程選擇：**
- 希望開始日期：_______
- 期望完成日期：_______
- 是否有彈性：是 / 否

**付款條件（建議）：**
- 一次付清
- 分期付款（3331 模式：簽約 30% / 中期 30% / 交付 30% / 驗收 10%）
- 其他方式（可說明）

### Step 9｜交付時你想拿到什麼？

**問法：** 「最後你想拿到哪些成果？」

**多選：** 原始碼 / 後台帳密 / 教學影片 / 維護服務 / 上線代辦

**溝通方式偏好：**
- 文字訊息（Line / Email）
- 語音通話
- 視訊會議
- 定期進度報告
- 其他（可說明）

### Step 10｜還有沒有其他補充？

**問法：** 「有沒有想特別強調或擔心的部分？」

**常見補充：**

- 擔心資料安全
- 怕操作太難
- 想之後能持續擴充功能
- 需要簽署保密協議（NDA）
- 版權歸屬需求
- 修改次數限制
- 其他特殊要求

## 三、互動設計建議

- 一次只顯示一題 → 減少焦慮。
- 每題附「看範例」「我不確定」「AI 幫我補」三個輔助按鈕。
- 題目右側即時預覽卡：「目前你描述的專案看起來像是——官網 + 預約系統」。
- 若回答過於模糊 → AI 自動追問「那你希望他用起來像什麼？」

**預算拉桿特殊設計：**
- 使用雙拉桿（Range Slider）設計，可同時設定最低與最高預算
- 拉桿拖曳時即時顯示數值，並顯示對應的專案規模提示（如：$40k–$80k → 中小型專案）
- 提供快速選項按鈕：$20k 以下 / $20k–$50k / $50k–$100k / $100k–$200k / $200k 以上
- 點擊快速選項後，拉桿自動調整到對應區間
- 若選擇「先估型」，顯示友善提示並建議從 MVP 開始

## 四、技術落地

**前端：** React + React Hook Form + Zod 驗證 + Framer Motion（逐題過場動畫）

**預算拉桿實作：**
- 使用 `rc-slider` 或 `@mui/material Slider` 實作雙拉桿（Range Slider）
- 整合 React Hook Form 的 `Controller` 進行表單驗證
- 使用 Zod Schema 驗證預算區間合理性（最低 < 最高）
- 拉桿數值格式化：超過 $100k 顯示為「$100k+」，小數點自動四捨五入
- 響應式設計：手機版使用觸控優化的拉桿尺寸

**後端：** PostgreSQL JSONB 結構儲存回答；AI 生成 `ai_summary` 用於搜尋。

**管理介面：** 可用於 A/B 測試問題順序與文案轉換率。

## 五、其他接案平台發案欄位參考

為確保 200 OK 的發案流程能涵蓋所有必要資訊，以下整理常見接案平台的發案欄位與步驟順序：

### 常見接案平台

- **國際平台：** Upwork、Freelancer.com、Fiverr
- **台灣平台：** CASE 1、Tasker 出任務、518外包、1111外包網、PRO360

### 標準發案欄位清單

| 欄位名稱 | 說明 | 對應 200 OK 步驟 | 必填 |
|---------|------|----------------|------|
| **專案標題** | 簡明扼要描述專案內容 | Step 1（自動生成） | ✅ |
| **專案分類** | 選擇專案類型（網站、App、系統等） | Step 1 | ✅ |
| **專案描述** | 詳細說明背景、目標、需求 | Step 2 + Step 3 + Step 4 | ✅ |
| **工作範圍** | 列出需完成的任務與功能 | Step 4 | ✅ |
| **技能要求** | 所需技術技能、經驗 | Step 7（自動轉譯） | ⚠️ |
| **預算範圍** | 預算區間或時薪 | Step 8 | ⚠️ |
| **時程安排** | 開始日期、里程碑、截止日期 | Step 8 | ⚠️ |
| **交付物** | 預期成果與檔案格式 | Step 9 | ✅ |

| **聯絡資訊** | 電話、Email、Line ID | （使用者帳號資訊） | ✅ |

### 各平台特色欄位

#### 國際平台（Upwork、Freelancer）

1. **專案類型**：固定價格 vs 時薪制
2. **經驗等級**：初級、中級、專家
3. **語言要求**：需要的溝通語言
4. **地點偏好**：本地、遠端、時區
5. **專案可見度**：公開 vs 僅邀請
6. **技能測試**：接案者是否需要通過技能測驗

#### 台灣平台（CASE 1、Tasker）

1. **案件性質**：一次性、長期合作、維護保養
2. **地點要求**：面談地點、工作地點
3. **緊急程度**：一般、急件、可談
4. **付款方式**：現金、轉帳、分期
5. **保密協議**：是否需要簽署 NDA
6. **作品集需求**：是否需要提供過往案例

### 建議發案流程步驟（標準化）

1. **前置準備**
   - 收集專案相關資訊
   - 確認預算與時程
   - 準備參考案例或文件

2. **選擇平台與分類**
   - 選擇合適的接案平台
   - 選擇專案分類標籤

3. **填寫基本資訊**
   - 專案標題（吸引人且具體）
   - 專案描述（清楚說明需求）
   - 工作範圍與功能列表

4. **設定條件**
   - 技能要求
   - 預算範圍
   - 時程安排

5. **補充細節**
   - 交付物清單
   - 付款條件（建議：3331 模式）
   - 溝通方式偏好

6. **附加資訊**
   - 參考案例連結
   - 特殊要求（保密、版權等）
   - 聯絡方式

7. **發布與管理**
   - 檢查所有資訊無誤
   - 發布專案
   - 回應接案者提問
   - 評估提案並選擇合作對象

### 付款條件建議（3331 模式）

建議採用分期付款以降低風險：

- **簽約 30%**：確定合作後支付
- **中期 30%**：完成重要里程碑時支付
- **交付 30%**：專案完成交付時支付
- **驗收 10%**：驗收通過後支付

### 200 OK 設計優勢

相比傳統平台，200 OK 的引導式設計有以下優勢：

1. **降低技術門檻**：不使用技術名詞，用生活化語言引導
2. **完整資訊收集**：透過分步驟引導，避免遺漏重要資訊
3. **自動結構化**：將回答自動轉換成結構化 JSONB，方便接案者理解
4. **AI 智能輔助**：自動生成摘要、建議功能、推薦風格
5. **視覺化預覽**：即時預覽專案描述，讓發案者確認資訊正確性

## 六、延伸功能構想

- **AI 輔助模板生成：** 根據回答自動生成可分享的 Project Brief（含 wireframe）。
- **常見案例建議：** 輸入「LineBot」時顯示過往成功案例與報價區間。
- **完成率誘因：** 當填完所有步驟可解鎖「預估報價」或「推薦接案者 Top 3」。
- **付款條件建議器：** 根據專案規模自動建議付款階段與比例。
